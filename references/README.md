# References

Reading notes and citation summaries live here. Entries are
added as readings are completed.

## How notes are organized

Each reading should have a short entry containing:

- **Summary.** A few sentences on what the reading argues or
  reports. Short is better than comprehensive.
- **Method notes.** What the study did, what it measured, and
  what it did not. Enough for a later reader to decide whether
  to go back to the source.
- **System-design implications.** Which design decisions this
  reading should influence in the repository, and why.
- **Limitations.** Known constraints on the reading — scope,
  sample, assumptions, conflicts of interest if relevant.
- **Relevance to repository tracks.** Which of Tracks 1–4 the
  reading relates to, and in what way.

Entries should be honest about what the reader (i.e., the
author of the note) did and did not understand.

## Rules

- No fabricated citations. If a reference is not verified, it
  is marked TODO or left out.
- A reading note is written after the reading is read, not
  before.
- Notes should be short. If a note is long, split it.

## Files

- `reading-list.md` — reading list organized by the same
  categories as `docs/literature-map.md`. Populated as entries
  are added.
