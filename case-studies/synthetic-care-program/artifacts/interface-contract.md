# Interface contract: synthetic care program dashboard

## Artifact

- Prototype: `synthetic-care-program`
- Interface name: synthetic care program dashboard
- Interface version: `SCP-001`
- Owner: repository prototype
- Last updated: 2026-07-06

## Safety boundary

This interface is a static synthetic research sketch. It is not
medical advice, not clinical decision support, and not a
validated workflow tool.

## Workflow context

- Intended user: repository reader evaluating workflow design.
- Workflow moment: synthetic review before governance decision.
- User action being studied: reading a generated risk display
  with limitations and auditability.
- Action this interface must not request: diagnosis, treatment,
  triage, escalation, discharge, or patient-specific action.
- Time pressure: illustrative only.
- Interruption level: passive display.

## Inputs

- Source artifact: `outputs/synthetic_cohort.csv`.
- Required fields: subject ID, severity, comorbidity, risk
  score, and risk band.
- Refresh cadence: regenerated by `run_case.py`.
- Missing input behavior: not modeled.
- Invalid input behavior: not modeled.

## Outputs

- Primary display: synthetic risk rows.
- Uncertainty display: limitation text rather than confidence
  claims.
- Explanation display: synthetic severity and comorbidity fields.
- Limitation display: visible dashboard safety boundary.
- Audit event: `outputs/workflow_audit_log.csv`.

## Human control

- Override path: represented as synthetic audit review only.
- Required reason capture: workflow hypothesis review.
- Accountability boundary: no real decision is supported.
- Escalation path: governance review in the safety case.

## Failure modes

- Known misuse: interpreting a generated risk row as clinical
  decision support.
- Alert fatigue risk: monitored through synthetic alert rate in
  later milestones.
- Over-trust risk: dashboard table may look operational.
- Under-trust risk: all outputs may look blocked because the
  project is intentionally conservative.
- Fallback behavior: do not use the interface for real
  decisions.

## Evaluation

- Usability check: not performed.
- Workflow burden check: audit log only.
- Audit completeness check: generated audit rows.
- Local validation precondition: real data, workflow, calibration
  and governance reviews would be required before any deployment
  conversation.
